Primeros pasos

Usuario

$ git config --global user.name "Maximiliano Danelutti"
$ git config --global user.email maximiliano@fizzmod.com

Por qué --global

Esto hace que toda configuración sea para el usuario actual del sistema

Extra pero útil

$ git config --global credential.helper 'cache --[options]'
$ git config --list

Empezar con un repositorio

Crear uno

$ git init

Obtener uno existente

$ git clone https://github.com/{user}/{repo}.git {alias}(opcional)

Trabajando con remotos (remote)

¿Qué es "origin"?

Es el remote de donde se obtuvo el repo, ademas es el nombre por default que asigna git

Ver remotes configurados

$ git remote

Ver remotes configurados + URLs

$ git remote -v

Agregar un remote

$ git remote add [nombre] [url]

Renombrar un remote

$ git remote rename [nombre-actual] [nombre-nuevo]

Ver mas información sobre un remote

$ git remote show [nombre]

Comandos básicos

Actualizar repositorio local con el del server

$ git pull [remote] [branch]

Agregar al área de preparación (Staging Area)

$ git add [archivo]

Confirmar lo que se encuentra en área de preparación

$ git commit -m "Mensaje del commit" | --amend

Actualizar el server con el contenido local

$ git push -u(opcional) [remote] [branch]

Actualizar repositorio local

$ git fetch [remote]

Estado actual de tu area de trabajo con respecto al server

$ git status (-s | --short)

Ver lineas modificadas o eliminadas

$ git diff (--staged/cached | --stat)

Ver historial de confirmaciones

$ git log (-p | -2 | --stat | --graph | --oneline | --decorate)

Volver archivos al estado inicial

$ git checkout .
$ git checkout -- file

Deshacer archivos preparados para confirmar

$ git reset HEAD .

Alias

En git se permite establecer tanto a los comandos simples como complejos alias y así evitar que sea necesario escribirlos por completo

Como declararlos

$ git config --global alias.co checkout
$ git config --global alias.br branch
$ git config --global alias.st status

Otros ejemplos

$ git config --global alias.unstage 'reset HEAD --'
$ git config --global alias.last 'log -1 HEAD'

Ramificaciones (Branches)

¿Qué es una rama (Branch)?

Es un puntero móvil apuntado a un commit particular

Como crear una

$ git branch [nombre-branch]

tener en cuenta que este comando solo crea la branch

Listar las branches

$ git branch (-v)

¿Qué es HEAD?

Es el puntero especial que utiliza git para saber en que branch estas actualmente de forma local

Como resultado de crear una branch "testing" vamos a obtener la siguiente estructura

Como realizamos el cambio de branch...

$ git checkout [nombre-branch]

Si el checkout es a "testing" va a ocurrir lo siguiente

repetir los pasos anteriores pero en un solo comando

$ git checkout -b [nombre-branch]

esta es la forma de resumir la creación de la branch y cambiar el HEAD a la nueva

Incorporar cambios de otra branch

$ git merge [nombre-branch]

este comando va a hacer un merge de [nombre-branch] en el HEAD actual

Ver branches que se encuentran o no en la actual (HEAD)

$ git branch --merged
$ git branch --no-merged

Ver en cuales se encuentra la actual

$ git branch --contains

Borrar banch

Localmente:

$ git branch -d [nombre-branch]
$ git branch -D [nombre-branch]

En el servidor (remote):

$ git push [remote] --delete [nombre-branch]
$ git push [remote] :[nombre-branch]

Trabajando Branch + Remote

$ git branch -u [remote]/[nombre-branch]
$ git branch --set-upstream-to [remote]/[nombre-branch]

Ignorar archivos

Si creamos el archivo ".gitignore" no permitirá ignorar todo lo declarado en el mismo

  • Las lineas en blanco o que comiencen con # son ignoradas

  • Acepta patrón glob estándar

  • Los patrones pueden terminar en barra para especificar directorios

  • Los patrones pueden ser negados con signo de exclamación (!)
