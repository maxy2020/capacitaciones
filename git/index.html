Primeros pasos

Usuario

$ git config --global user.name "Maxx"
$ git config --global user.email maximiliano@fizzmod.com

Por qué --global

Esto hace que toda configuración sea para el usuario actual del sistema

Extra pero útil

$ git config --global credential.helper cache
$ git config --list

Empezar con un repositorio

Crear uno

$ git init

Obtener uno existente

$ git clone https://github.com/{user}/{repo}.git {alias}(opcional)

Trabajando con remotos (remote)

¿Qué es "origin"?

Es el remote de donde se obtuvo el repo, ademas es el nombre por default que asigna git

Ver remotes configurados

$ git remote

Ver remotes configurados + URLs

$ git remote -v

Agregar un remote

$ git remote add [nombre] [url]

Renombrar un remote

$ git remote rename [nombre-actual] [nombre-nuevo]

Comandos básicos

Actualizar repositorio local con el del server

$ git pull [remote] [branch]

Agregar al área de preparación (Staging Area)

$ git add .

Confirmar lo que se encuentra en área de preparación

$ git commit -a(opcional) -m "Mensaje del commit" | --amend

Actualizar el server con el contenido local

$ git push -u(opcional) [remote] [branch]

Actualizar repositorio local

$ git fetch [remote]

Estado actual de tu area de trabajo con respecto al server

$ git status (-s | --short)

Ver lineas modificadas o eliminadas

$ git diff (--staged/cached | --stat)

Ver historial de confirmaciones

$ git log (-p | -2 | --stat | --graph | --oneline | --decorate)

Volver archivos al estado inicial

$ git checkout .
$ git checkout -- file

Deshacer archivos preparados para confirmar

$ git reset HEAD .

Alias

En git se permite establecer tanto a los comandos simples como complejos alias y así evitar que sea necesario escribirlos por completo

Como declararlos

$ git config --global alias.co checkout
$ git config --global alias.br branch
$ git config --global alias.st status

Otros ejemplos

$ git config --global alias.unstage 'reset HEAD --'
$ git config --global alias.last 'log -1 HEAD'
